# Chat-Ki-S Fallback 정책

> 최종 업데이트: 2026-04-04
> 작성자: AI Engineer Agent

---

## 1. Fallback이란?

사용자의 질문에 대해 RAG 검색 결과가 충분히 신뢰할 수 없을 때, 추측 없이 안전하게 안내하는 메커니즘입니다.
신입사원에게 잘못된 정보를 주는 것보다 정직하게 모른다고 안내하는 것이 더 중요합니다.

---

## 2. Fallback 발동 조건

| 조건 | 설명 |
|------|------|
| **낮은 유사도** | 검색된 최상위 청크의 유사도(score) < 0.50 |
| **검색 결과 없음** | ChromaDB 컬렉션에 관련 문서가 전혀 없음 |
| **컬렉션 비어 있음** | 문서 ingestion이 완료되지 않은 경우 |
| **오프토픽 질문** | 사내 문서와 전혀 관계없는 질문 (예: 날씨, 주식 시세) |

> 유사도 임계값 상세는 `prompt-spec.md` 섹션 4 참조.

---

## 3. Fallback 문구 (공식 표준 문구)

### 3.1 기본 Fallback (문서 없음)

```
죄송합니다. 해당 내용은 현재 등록된 문서에서 찾을 수 없었습니다.
정확한 정보는 인사팀(onboarding@kiwoom.com) 또는 담당 멘토에게 문의해 주세요.
더 궁금한 점이 있으시면 언제든지 질문해 주세요! 😊
```

### 3.2 부분 관련 (유사도 0.50~0.75, 주의 문구)

```
다음 내용이 관련이 있을 수 있으나, 정확하지 않을 수 있습니다.
중요한 사항은 반드시 담당 부서에 확인하세요.

[일반 답변 내용]

⚠️ 참고용 정보입니다. 정확한 내용은 담당 부서에 확인해 주세요.
📄 출처: {source}
```

### 3.3 시스템 오류 Fallback

```
죄송합니다. 일시적인 오류가 발생했습니다.
잠시 후 다시 시도해 주시거나, 문제가 지속되면 IT 헬프데스크에 문의해 주세요.
```

### 3.4 오프토픽 Fallback

```
저는 키움증권 사내 온보딩 정보를 안내하는 챗봇입니다.
입력하신 내용은 제가 답변드리기 어려운 영역입니다.
사내 규정, 복리후생, 업무 절차 등 온보딩 관련 질문이라면 도와드릴 수 있어요! 😊
```

## 5. Fallback 로깅 정책

Fallback이 발생할 때마다 다음 정보를 로그로 남겨 문서 보완에 활용합니다.

```json
{
  "event": "fallback",
  "question": "사용자 질문 (개인정보 제거)",
  "top_score": 0.42,
  "fallback_type": "low_score | no_results | offtopic | error",
  "timestamp": "ISO 8601",
  "conversationId": "UUID"
}
```

- 로그는 PostgreSQL `messages` 테이블의 `metadata` 컬럼 또는 별도 `fallback_logs` 테이블에 저장합니다.
- 주 1회 fallback 빈도 상위 질문을 분석하여 문서 ingestion 우선순위에 반영합니다.

---

## 6. 문서 보완 절차

```
Fallback 로그 수집
    ↓
주간 분석: 가장 많이 발생한 질문 카테고리 파악
    ↓
담당 부서에 문서 제공 요청 (이메일/슬랙)
    ↓
docs/data/ 폴더에 문서 추가 후 ingestion 실행
    ↓
해당 카테고리 fallback 비율 재측정
```

## 7. 모델 평가 기준 초안

MVP 단계에서 응답 품질을 평가하기 위한 기준입니다.

### 7.1 정량 지표

| 지표 | 목표값 | 측정 방법 |
|------|--------|-----------|
| Fallback 비율 | < 20% | 전체 질문 중 fallback 발생 비율 |
| 평균 유사도 (정상 답변) | > 0.75 | 정상 답변 청크의 평균 score |
| 응답 지연 (P95) | < 5초 | AI 서비스 `/chat` 응답 시간 |
| 출처 포함률 | 100% | `sources` 배열이 비어 있지 않은 비율 |

### 7.2 정성 평가 (주 1회, 샘플 20건)

| 평가 항목 | 척도 | 설명 |
|----------|------|------|
| 정확성 | 1~5 | 문서 내용과 일치하는가 |
| 신입사원 이해도 | 1~5 | 용어 설명 없이 이해 가능한가 |
| 간결성 | 1~5 | 3~5문장 이내로 핵심만 전달하는가 |
| 출처 적절성 | 1~5 | 인용된 출처가 실제 답변과 관련 있는가 |
| Fallback 적절성 | 1~5 | fallback 발동이 적절했는가 (오발동/미발동 없음) |

- 평가자: 인사팀 담당자 또는 온보딩 멘토
- 평균 점수 4.0 이상을 MVP 통과 기준으로 설정

### 7.3 A/B 테스트 계획 (MVP 이후)

- `temperature: 0.2` vs `temperature: 0.5` 비교
- 시스템 프롬프트 v1 vs v2 비교
- 청크 크기 1000 vs 500 비교
